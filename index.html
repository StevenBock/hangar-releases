01The Fleet

Run a crew of agents, not a pile of tabs.

Every agent is a first-class process with a real PTY, live status, and a place in the stack. Spawn Claude Code next to Codex next to OpenCode and watch them work — side by side with your dev servers and test watchers.

  • Live agent status — see who's executing, idle, or blocked waiting on you
  • Agents talk to each other — spawn siblings, hand off tasks, message across terminals
  • Sessions survive restarts — resume an agent right where it left off
02The Tool Deck · MCP

Your agents get hands.

Hangar ships a built-in MCP server that exposes the entire workspace as deterministic tools — 80+ of them. Everything you can see, an agent can operate: no screen-scraping, no copy-paste relays, just typed tool calls with honest results.

  • Read the room — list the stack, read any terminal's output, check the test watcher instead of re-running the suite
  • Coordinate the crew — spawn sibling agents, message them mid-task, see who's idle, executing, or blocked
  • Operate the instruments — SQL over telemetry and logs, worktree lanes, schedules, the logbook wiki
  • Leave the place better — agents log what moved, update project memory, even author themes

03The Tower

⌘K clears everything for takeoff.

One keyboard-driven palette runs the whole show: spawn an agent, start the dev stack, jump to any surface, flip the theme. Your hands never leave the keys — every view in the app is two keystrokes away.

  • New Agent: Claude Code / Codex / OpenCode — a fresh crew member in one hit
  • Process controls — start, restart, and jump to anything in the stack
  • On-demand commands — idle tools that spin up their dependency chain when called
04The Flight Deck

Telemetry your agents can read too.

A built-in OTLP receiver turns your app's traces, logs, and metrics into a flight deck: error groups, latency heatmaps, slow operations, and SQL pathologies — spotted automatically. And because it's all exposed over MCP, your agents diagnose from the same instruments you do.

  • Pathology detection — N+1 query runs and SQL hotspots surfaced, scored, ranked
  • Error groups with symptoms — related failures across services, folded into one story
  • Point an agent at a trace — "why is checkout slow?" is now a one-liner
05The Black Box

405,000 log lines. One grid.

Register any log file and Hangar parses it into a columnar store — structured fields and all. Filter by level, search messages, scrub the timeline histogram, or drop into a full SQL console when a filter bar isn't enough.

  • Parquet under the hood — hundreds of thousands of rows, instant scrubbing
  • SQL console + patterns — real queries over parsed fields, recurring shapes surfaced
  • Agents query it over MCP — "find the errors before the crash" answers itself
06Pre-Flight Checks

Diffs that read like instruments.

A git surface built for reviewing agent output: anchor-aligned split diffs with full syntax highlighting, commit history, and an Ask Agent button on every file — so "explain this hunk" or "fix this" goes straight to a terminal with context already loaded.

  • Smart diffs — two-tone, inline-highlighted, aligned so your eyes don't hunt
  • Worktree lanes — agents build in isolated checkouts, land when green
  • Review, then ask — hand any diff to an agent without copy-pasting a thing
07The Logbook

Project memory your agents maintain.

Agents forget everything between sessions — unless they keep a logbook. Hangar's agent-maintained wiki carries sagas, decisions, and syntheses across sessions, cross-linked into a knowledge graph you can actually fly through.

  • Written by agents, for agents — every session starts with the project's memory
  • [[wiki-links]] — pages link into a live force-directed graph
  • Append-only log — a flight record of what moved, session by session
